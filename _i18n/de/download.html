OmegaT kann auf jedem Betriebssystem installiert werden, das die Java Runtime Environment Version 11 ausführen kann.

Wählen Sie, ob Sie OmegaT mit oder ohne Java Runtime Environment (JRE) herunterladen möchten. Wenn auf Ihrem Windows-System schon eine geeignete JRE installiert ist, wählen Sie "ohne JRE". Falls dies nicht der Fall ist oder wenn Sie sich nicht sicher sind, wählen Sie "mit JRE".

OmegaT wird normalerweise auf jedem Linux-System laufen, auf dem eine geeignete Version der Java Runtime Environment (JRE) installiert werden kann. OmegaT benötigt derzeit JRE Version 11 oder höher. Um herauszufinden, welche Version der JRE, falls vorhanden, auf Ihrem Linux-System installiert ist, öffnen Sie eine Konsole (Befehlszeilenfenster) und schreiben Sie:

$ java -version

Wenn Sie sich nicht sicher sind, ob auf Ihrem System bereits eine geeignete JRE installiert ist, können Sie trotzdem eine OmegaT-Version mit JRE herunterladen und installieren. Diese Version der JRE wird nur lokal für OmegaT verwendet und steht weder in Konflikt mit anderen JRE-Installationen, noch wird sie diese überschreiben oder deinstallieren und kann dementsprechend bedenkenlos verwendet werden.

Legacy-Versionen

Notfalls können Sie immer noch veraltete Versionen von OmegaT beziehen.

Legacy-Versionen sind veraltete Versionen. Für die meisten Benutzer sind diese nicht von Interesse. Einige Benutzer könnten jedoch auf Hindernisse stoßen (auferlegt durch ihr Betriebssystem oder ihre Java-Version), wodurch sie gezwungen sind, eine ältere Version zu verwenden.

Mit/ohne JRE

OmegaT kann nur ausgeführt werden, wenn die Java Runtime Environment (JRE) auf Ihrem Computer installiert ist.

Wenn Sie sich sicher sind, dass eine geeignete Version der JRE bereits auf Ihrem Computer installiert ist, können Sie eine Version von OmegaT ohne die JRE herunterladen. Der Download geht schneller vonstatten, und die Installation benötigt weniger Speicherplatz auf Ihrer Festplatte. Wählen Sie in diesem Fall eine Version "ohne JRE".

Wenn Sie die OmegaT-Version mit JRE installieren, obwohl die JRE bereits auf Ihrem Computer installiert ist, wird die JRE von OmegaT separat installiert, ohne in Konflikt mit anderen JRE-Installationen zu geraten. Es ist daher immer sicher, die Version von OmegaT mit JRE zu installieren. Wenn Sie sich nicht sicher sind, ob Sie eine geeignete Version der JRE haben, wählen Sie eine Version "mit JRE" (oder die Standardversion).

Linux-Benutzer: Die meisten Linux-Distributionen werden zusammen mit einer Java Runtime Environment bereitgestellt, doch einige Open-Source-Implementierungen kommen mit OmegaT nicht richtig zurecht (auch wenn dies vielleicht nicht sofort ersichtlich ist). Wenn Sie sich noch nicht so gut mit Linux auskennen, raten wir Ihnen, eine OmegaT-Version zu wählen, die die JRE enthält. Erfahrene Linux-Benutzer bevorzugen es eher, die JRE von Oracle (früher Sun) direkt auf Ihr System zu installieren.

Notarisiert vs Unsigniert

macOS versucht, die Sicherheit zu erhöhen, indem es die Ausführung nicht verifizierter Programme erschwert.

Für macOS gibt es sowohl eine notarisierte als auch eine unsignierte Version von OmegaT; beide Versionen sind funktional identisch, wobei die notarisierte Version vorzuziehen ist, da das System auf diese Weise erkennt, dass sie a) die Veröffentlichung eines bekannten Entwicklers ist und b) von Apple gescannt wurde.

Es kann zu einer Verzögerung bei der Veröffentlichung der notarisierten Version kommen. Benutzer, die es eilig haben, können stattdessen die unsignierte Version verwenden; wenn Sie Schwierigkeiten mit dem Starten der App haben, schauen Sie sich Wie man eine App von einem unbekannten Entwickler öffnet an.

Die Master- bzw. Alpha-Version spiegelt den neuesten Stand der Entwicklung wider. Sie wird nur ernsthaften Entwicklern und Testern empfohlen. Sie kann auch mit dem git-Befehl abgerufen werden:

git clone https://git.code.sf.net/p/omegat/code omegat-code

Regelmäßig erscheinende Builds sind auf SourceForge verfügbar.
