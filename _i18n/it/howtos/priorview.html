Guida: L'utilità di anteprima "Priorview"

Premesse e principi

Priorview è una piccola utilità che fornisce una funzione di anteprima per OmegaT. Per le premesse e i principi, consultare la Guida all'anteprima. Questa guida descrive l'uso di Priorview per Windows e Linux. Se si utilizza Mac OSX, è possibile scaricare una versione di Priorview per Mac qui. La versione Mac OSX non è ancora documentata e le seguenti informazioni non si applicano ad essa.

Software richiesto

Per utilizzare Priorview su Windows o Linux, è necessario quanto segue:

1. Un browser, in particolare:

  • Opera
    oppure
  • Firefox insieme con il componente aggiuntivo "ReloadEvery" per Firefox
    Installare prima Firefox (se non è già installato), poi ReloadEvery.
  • (Scaricare dai collegamenti forniti; gli utenti Linux potrebbero preferire installarlo dagli archivi della loro distribuzione, laddove questo sia possibile)

2. Un'utilità di conversione, in particolare:

  • Abiword
    oppure
  • LibreOffice insieme con il plug-in Unoconv

3. Tcl/Tk

Quasi tutte le versioni attuali di Tcl/Tk saranno probabilmente adatte, ma bisogna avere sia Tcl (l'interprete di script), sia Tk (il componente grafico).

4. Priorview

Per installare Priorview, basta scompattare l'archivio zip in qualsiasi luogo adatto sul tuo disco rigido. Per il lancio di Priorview, aprire un prompt dei comandi/terminale/console e digitare:

cd percorso

dove percorso è il percorso completo della cartella in cui è stato memorizzato preview.tcl. Digitare poi:

wish priorview.tcl

È possibile semplificare l'avvio scrivendo un file di script, cioè un file di testo contenente queste due righe.

Usare Priorview

Se tutto va bene, Priorview dovrebbe comparire, una volta avviato come descritto sopra. In esso, selezionare:

  • il nome del file di destinazione (in /target nel proprio progetto OmegaT);
  • il browser che si è installato;
  • la frequenza con cui lo script deve verificare se questo file è stato modificato (intervallo di polling);
  • l'utilità di conversione file che si è installato.

Dopo aver selezionato tutti questi elementi, apparirà il pulsante di Start. Quando si aziona, il comando avvia il browser con una pagina di vuota, avvisando l'utente di impostare la funzione di aggiornamento automatico, che dovrà essere fatto, e poi di premere Ctrl+S, Ctrl+D in OmegaT in modo da avviare l'aggiornamento automatico del browser.

Durante l'esecuzione di Priorview, è possibile:

  • interromperlo e riavviarlo
  • selezionare un nuovo file (se si lavora in un progetto contenente diversi file, non si deve riavviare Priorview quando si raggiunge la fine di un file: basta semplicemente selezionare quello successivo);
  • modificare l'intervallo di polling.

Una volta avviato Priorview, tuttavia, non è possibile selezionare un browser o un utilità di conversione differente; queste opzioni sono dunque nascoste dopo l'avvio. Per selezionare un browser o un'utilità di conversione diversa, è necessario chiudere Priorview e riavviarlo.

Il pulsante Esci chiude Priorview ed elimina anche i file temporanei creati (altri metodi di chiusura di Priorview non elimineranno i file temporanei).

Risoluzione dei problemi

Se il percorso della posizione del priorview.tcl contiene spazi, metterlo tra virgolette, ad esempio:

cd "C:\Miei file\priorview"

Se Priorview non si avvia:

A seconda della versione di Tcl/Tk installata, potrebbe essere necessario aggiungere il numero di versione per il comando wish, ad esempio:

wish8.5 priorview.tcl

Prestare attenzione alle maiuscole/minuscole.

Se il file visualizzato nel browser non viene aggiornato, controllare quanto segue:

  • Di essersi ricordati di aver premuto Ctrl+S, Ctrl+D in OmegaT. Se non lo si fa, non ci sarà alcuna modifica da individuare/mostrare!
  • Che l'utilità di conversione di file sia installata correttamente e stia funzionando. Questo è possibile verificarlo con l'appropriato comando nella riga di comando (terminale/console/prompt di DOS). In ogni caso, il file è il nome di un file con estensione .docx o .odt. Se tutto è corretto, sarà prodotta una versione .html del file:
    Per Abiword:
    abiword --to=html file
    Per LibreOffice+Unoconv:br /> unoconv -f html file.
  • Che si sia impostato l'intervallo di aggiornamento automatico nel browser.
